UI/UX Reflection — Nielsen Heuristics

The heuristic that feels most aligned with Don Norman's ideas in Chapter 1 is visibility of system status. Norman argues that users need clear, immediate feedback so they can understand what their actions are doing. Similarly, Jakob Nielsen emphasizes that systems should always keep users informed about what is happening. Both are concerned with making interactions understandable and reducing confusion.

The overlap between the two readings is that feedback is essential to user experience. Norman explains that people build mental models based on what they can see, and when feedback is unclear or overwhelming, those models break down. Nielsen turns that idea into a more direct principle by saying systems should always show their current state in a clear and timely way. In my project, this connection is clear. When the user presses a key, their message grows, which acts as feedback. However, it fades quickly and gets drowned out by louder system messages. Even though the system is responding, the feedback is not clear or stable enough to help the user understand what is happening. This creates a disconnect between the user's action and their understanding, which is exactly the kind of issue both Norman and Nielsen warn about.

One heuristic my project struggles with is user control and freedom. Once the interaction starts, the user is basically stuck as the system becomes more chaotic. There is no way to undo actions, pause the experience, or reset it. This happens as soon as the user starts pressing keys and the system messages begin to multiply across the screen. This matters because it removes the user's sense of control. Even though the concept is about being overwhelmed, the lack of control can feel frustrating in a way that is not meaningful. Users might stop interacting not because they understand the idea, but because they feel stuck.

Another heuristic it struggles with is aesthetic and minimalist design. The screen quickly fills up with repeated “I SPEAK LOUDER” messages, which makes it difficult to focus on the user's own text. This happens where multiple system messages are generated with each key press. While this is intentional, it creates a lot of visual noise. This matters because when everything is competing for attention, the main idea can get lost. Instead of clearly showing the contrast between the user and the system, the experience can become hard to read.

For the sketchbook itself, I think it most clearly violates recognition rather than recall. There are no instructions telling the user what to do, so they have to figure out on their own that pressing any key triggers the interaction. This forces users to guess instead of recognizing available actions. There is also no clear indication of progress, such as how many key presses are needed to reach the breakthrough moment. Users have to rely on subtle changes like text size or sound, which makes the experience harder to understand. There is also an issue with visibility of system status. The system is clearly changing through more messages, louder sound, and larger text, but it does not clearly communicate what those changes mean. Users might not realize they are progressing toward a final state, which makes the interaction feel random instead of intentional.

The hardest heuristic to implement honestly is aesthetic and minimalist design. The main reason is that my project is built around the idea of overload and noise. If I simplify the visuals too much, I lose the feeling of being overwhelmed, which is the core concept. If I do not simplify enough, the interface becomes too chaotic to understand. This makes it more of a design tradeoff than a technical problem. I want users to feel overwhelmed, but I also want them to understand what that feeling represents. Balancing those two goals is difficult, and it shows how following heuristics isn't always straightforward. Sometimes breaking them is part of making the concept work, but it still requires careful decision making so the experience remains meaningful rather than confusing.

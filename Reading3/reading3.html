Reading Reflection 3

Which rule feels most absent in your sketchbook interface?

The rule that feels most absent is “Keep users in control.” In this interaction, the user presses a key, but the system quickly overwhelms them. Each press increases noise, adds more system messages, and causes the user's own message to fade. Even though the user is interacting, it does not feel like they are actually in control. Instead, the system dominates the experience. The user also cannot clearly predict what will happen next. Their action leads to multiple effects at once, which makes the system feel chaotic. While the final “breakthrough” moment restores a sense of control, most of the interaction the user feels reactive rather than in charge.

Which rule would most improve your sketchbook interface?

The rule that would most improve the interface is “Offer informative feedback.” Right now, the system does respond to the user, but the feedback is hard to understand. The user sees their text grow and fade while more messages appear, but it is not clear what this means or how it connects to progress. Clearer feedback would help users understand that their actions are building toward something. For example, showing a sense of progression or stages would make the experience feel more intentional instead of random. The interaction would still feel overwhelming, but users would better understand why it is happening.

Which rule is hardest to implement?

The hardest rule to implement is “Prevent errors.” In this project, confusion is part of the concept. The system is meant to feel overwhelming and difficult, which makes it hard to define what an “error” even is. If a user stops interacting or misunderstands what to do, the system does not guide them or correct them. Making the system more error-proof would require adding clarity and guidance, but that could weaken the intended feeling of struggle. There is a balance that needs to be considered between making the interface usable and keeping the concept intact.

Did any rule contradict how you designed your system originally?

Yes, “Design dialogs to yield closure” contradicts the way I originally designed the system. This rule suggests that interactions should have a clear beginning, middle, and end but in this design, the experience is more ambiguous. While there is a “breakthrough,” it is not clearly presented. The design leans toward confusion and lack of structure, which goes against the idea of clarity and closure. The only closure is when the user reaches the final state and cannot click anymore. If I made the stages more visible to adhere to this rule better it could help provide better clarity but it could also reduce the meaning I was intending to convey. I feel that there are some rules I can break for the purpose of my skech while following others.

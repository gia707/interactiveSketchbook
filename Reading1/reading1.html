A Reflection on Chapter 1 of "The Design of Everyday Things" by Don Norman

My micro-project is about what it feels like to be silenced, using interaction to show that experience. The user tries to “speak” by pressing a key, but their message fades out really quickly while a bunch of aggressive system messages take over the screen. As they keep pressing, their message gets bigger, but the system also gets louder and more overwhelming. Eventually, if they keep going, there's a breaking point where their message finally takes over.

The main affordance in my project is pressing a key. That's the only action the user can take, and it represents speaking. Unlike before, this is now clearly signified through the instruction text on the page that says “press any key to try to speak.” So if someone encounters the project, they know what to do right away. However, even though the action itself is clear, what that action actually does is not. The system doesn't explain the consequences, so the affordance is obvious but the outcome is still unclear. This creates a situation where the user understands how to act, but not what will happen when they do.

Feedback is a big part of the experience. Every time the user presses a key, their text shows up again and gets bigger, so there is immediate visual feedback. At the same time, the system floods the screen with “I SPEAK LOUDER” messages and the crowd noise gets louder. So the system is definitely responding, but not in a helpful way. At first, the feedback is readable, but it quickly becomes overwhelming. There's a point in the middle where the screen is so crowded that it's hard to tell what's going on. The user might not know if they're making progress or just adding to the chaos. This is probably where the design is weakest in terms of clarity. Someone like Don Norman would probably say the system fails to clearly communicate its state at this point.

The mapping between input and outcome is kind of conflicted. Pressing a key makes your message bigger, which feels like progress, but it also increases the amount of noise around you. So one action is causing two opposite effects. That's not very intuitive, and it can make the user hesitate or question what they should do next. This confusion is intentional. I wanted the user to feel unsure about whether continuing is helping or hurting, since that reflects real situations where speaking up can lead to more resistance instead of being heard.

There aren't strict constraints in the system since the user can press keys as much as they want. But other types of constraints show up over time. The screen gets really crowded, messages overlap, and the sound gets louder. These act like invisible constraints that make it harder to keep interacting. So even though nothing is physically stopping the user, the experience itself becomes limiting. Instead of guiding the user, the system kind of works against them, which fits the theme of being silenced.

If I had to point out the biggest design weakness, it's the lack of clarity once the system becomes overwhelming. The feedback stops being easy to understand, and the user can lose track of whether their actions are doing anything meaningful. From a usability perspective, that's a problem. But in this case, it's also intentional. I'm basically breaking those design rules to create that feeling of frustration, and then resolving it at the end when the message finally takes over and everything becomes clear again.

My biggest takeaway from The Design of Everyday Things is that design isn't just about making things work, it's about shaping how people behave. Earlier in the semester we talked about interaction design as something that can influence behavior on a bigger level, and I see that more clearly now. In my project, I'm not just letting the user press a key. I'm putting them in a situation where they have to decide if they want to keep going even when it feels pointless. The system is frustrating and overwhelming on purpose, but it also rewards persistence in the end. So the interaction is really about pushing the user through that experience, not just completing an action. In short, it's about perserverance and overcoming a system that seems designed to stop you.

I liked the reading, I thought it was very insightful and I think it will be really helpful for my future projects. I think the concepts of affordances, feedback, and mapping are really important to keep in mind when designing interactions, and I want to try to use them more intentionally in my future projects as a designer.

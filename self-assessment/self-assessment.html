Core Rule & System Clarity (20/20)

The core rule of my system is that every time the user presses a key, both a player message and a system message are generated, but the system message is always more visible (bigger and lasts longer). I kept this rule consistent throughout the project, so the imbalance is clear every time you interact with it.

State Progression (15/15)

The system moves through clear stages: starting empty, then showing both messages, then gradually becoming dominated by system messages as they stack and persist. Each stage changes how the user experiences the system, especially in terms of visibility and control.

Interaction & Evolution (15/15)

User interaction directly affects the system, by showing the progression of the player overcoming the system's oppression through persistent key presses. Over time, player messages disappear faster while system messages build up, so the experience evolves into something more overwhelming and frustrating but also more empowering as the player's mesage size grows bigger and more visible despite the system's attempt to overwhelm the player by the increased amount of system messages and their slight increase in size. The player can feel empowered as their message size increases and eventually overcomes the system's messages. This creates a clear sense of progression.

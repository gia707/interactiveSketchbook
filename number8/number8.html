Movement between imgs is determined by pressing the left or right arrow key which makes the currentIndex go either up or down by 1. Right arrow goes up by 1 and Left arrow goes down by 1. CurrentIndex is like recipe holder that flips and shows what recipe ur on. I think it's called a Rolodex. So the image shown is like the recipe u stop on in a rolodex. The system handles boundaries by creating a loop so it never goes out of range of the currentIndex. So if you go past the last img it just goes back to the first and vice versa. I learned that currentIndex is the state variable so it displays what img you are on and that navigation is just changing a number.

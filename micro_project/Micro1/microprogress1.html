CONCEPT SUMMARY:

This interaction, the user presses a key to make a message appear. At the same time, a system message also appears. The user's message fades quickly, while the system message is larger and fades more slowly. This creates a visible imbalance and portrays the user's input as less than the System's response.

PRIMARY VARIABLE

playerOpacity: the opacity of the player's message.

When a key is pressed, playerOpacity becomes 255. In the draw() function, the number slowly decreases. It controls how visible the player's message is. As the number gets smaller, the message fades away. WHAT I BUILT

week I built a simple system with one variable (playerOpacity) and one input (pressing any key) that have two visible states. State 1: No key pressed. The screen is empty (or background only). No messages are visible. State 2: Key is pressed. Two messages appear: A small white player message that fades quickly. A larger red system message that fades slower. The contrast between fading speeds creates the imbalance of visibility and inequality.

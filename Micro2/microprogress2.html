CONCEPT SUMMARY: My current interaction lets the user press any key to trigger two responses; a small white message that represents the player's input and a larger red message that represents the system's response. Every time a key is pressed the player's message appears and fades quickly, while the system's message appears and fades more slowly. This creates a visible imbalance where the player's input is less visible than the system's response, portraying an oppresive state. Despite the player's message fading quickly, it grows slightly larger with each key press. The main variable controlling change is the playerSize, which increases by 5 pixels every time a key is pressed. When playerSize becomes large enough, the state of the system changes and the breakthrough occurs where the player's voice becomes heard and overcomes the system's messages. This new addition of the playerSize variable and the breakthrough state creates a sense of progress and empowerment for the player, as they can see their input growing stronger and eventually overcoming the system's response, creating a sense of hope and resilience to the narrative of oppression.
